# 관리자 CSV 일괄 업로드 설계 (화면 13. 관리자 - CSV 일괄 업로드)

> **2026-07-21 실제 구현 완료.** 아래 설계 그대로 구현됨 — `app/src/app/api/admin/golf-courses/upload/route.ts`, `app/src/components/CsvUploadForm.tsx`. 응답 스펙(`totalRows`/`successCount`/`failCount`/`errors`, 오류 목록 상위 200건 제한)도 이 문서 그대로. 미검증(로컬 DB 접근 불가로 샌드박스에서 실제 업로드 동작 확인 못함) — 사용자가 로컬에서 지난번 만든 `wolsongricc-par.csv` 업로드해보고 결과 확인 필요.

골프장 루프별 9홀 Par(규정타수)를 CSV로 한 번에 등록/수정하기 위한 처리 로직 설계.
관련 화면: `doc/pages.md` 11~13번. 관련 모델: `GolfCourse`, `GolfCourseLoop`, `GolfCourseHole`, `User.role`.

**2026-07-16 갱신**: `GolfCourseLoop` 도입에 따라 CSV 포맷에 `루프명` 컬럼이 추가되고, 홀번호 범위가
1~18(골프장 전체)에서 1~9(루프 내부)로 바뀌었다. 골프장에 루프가 없던 구버전 데이터는 재작업 필요.

## CSV 포맷

```
골프장명,루프명,홀번호,Par
레이크사이드 CC,전반,1,4
레이크사이드 CC,전반,2,5
레이크사이드 CC,전반,3,3
레이크사이드 CC,후반,1,4
레이크사이드 CC,후반,2,4
...
```

- 첫 행은 헤더로 간주하고 파싱에서 제외
- 인코딩: UTF-8 (BOM 포함/미포함 모두 허용)
- `루프명`: 공백 불가. 골프장 내에서 유일해야 함(`@@unique([golfCourseId, name])`)
- `홀번호`: 1~9 사이 정수만 허용 (루프 내부 홀 번호)
- `Par`: 3/4/5 중 하나만 허용
- 파일 크기 제한: 1MB, 최대 2,000행 (9홀 × 루프 2~3개 × 약 80개 골프장 규모)

## API

`POST /api/admin/golf-courses/upload` (multipart/form-data, CSV 파일 1개)

### 처리 순서

1. **권한 검사**: 세션의 `role !== ADMIN`이면 403 응답, 화면 진입 자체는 `AdminGuard`가 서버에서 먼저 차단
2. **파싱**: 파일 크기/행 수 제한 검사 → 행 단위로 `골프장명,루프명,홀번호,Par` 파싱
3. **행 단위 유효성 검사**
   - 골프장명 공백 → 오류(`골프장명 누락`)
   - 루프명 공백 → 오류(`루프명 누락`)
   - 홀번호가 1~9 범위 밖 → 오류(`홀번호 범위 오류`)
   - Par가 3/4/5가 아님 → 오류(`Par 값 오류`)
4. **골프장 매칭**: `GolfCourse.name`과 정확히 일치하는 레코드 검색 (앞뒤 공백 trim 후 비교)
   - 매칭되는 골프장이 없음 → 오류(`골프장명 불일치`), 신규 골프장을 자동 생성하지는 않음 (오탈자 방지 목적 — 관리자가 이름 확인 후 재업로드)
   - 동일 이름의 골프장이 2건 이상 존재 → 오류(`골프장명 중복, 특정 불가`) — 추후 골프장 선택 UI(주소 병기)로 보완 예정
5. **루프 매칭/생성**: 매칭된 골프장 기준 `GolfCourseLoop.name`으로 검색, 없으면 자동 생성(`sortOrder`는 해당 골프장의 기존 루프 수를 이어서 부여)
6. **저장**: 매칭/생성된 루프 기준 `GolfCourseHole` upsert (`@@unique([loopId, holeNumber])` 제약 활용, 같은 루프·홀번호가 CSV에 중복 등장하면 마지막 값으로 덮어씀)
7. **처리 단위**: 파일 전체를 하나의 트랜잭션으로 묶지 않고 행 단위로 개별 처리 → 일부 행에 오류가 있어도 나머지 정상 행은 반영되는 부분 성공 허용
8. **응답**:
   ```json
   {
     "totalRows": 24,
     "successCount": 22,
     "failCount": 2,
     "errors": [
       { "row": 15, "courseName": "파인빌리 GC", "loopName": "전반", "message": "골프장명 불일치" },
       { "row": 19, "courseName": "그린밸리 CC", "loopName": "후반", "message": "Par 값 오류(6)" }
     ]
   }
   ```
   → 화면(13번)에서 성공/실패 건수 요약 + 실패 행 목록 테이블로 표시

## CSV 내보내기 (2026-07-21 추가)

업로드의 반대 방향 — DB에 이미 저장된 `GolfCourseLoop`/`GolfCourseHole` 데이터를 위와 동일한 포맷(`골프장명,루프명,홀번호,Par`)으로 내보내는 기능. 11번 화면 목업/설계에는 없던 항목이나, 사용자가 12번(루프·Par 입력) 작업 완료 후 "DB에 저장된 골프장별 코스 Par 정보를 CSV로 조사해서 저장해달라"고 요청해 추가.

- **API**: `GET /api/admin/golf-courses/export` (관리자 세션 필요, `requireAdminSession` 재사용)
- **응답**: `Content-Type: text/csv; charset=utf-8`, `Content-Disposition: attachment; filename="golf-course-par-export_YYYY-MM-DD.csv"` — 브라우저에서 직접 열면 자동 다운로드
- **내용**: 골프장명 오름차순 → 루프 `sortOrder` 오름차순 → 홀번호 오름차순. **아직 Par가 저장되지 않은 홀은 제외**(12번 화면 UI는 미저장 홀을 Par 4로 기본 표시하지만, 이 내보내기는 실제 DB 값만 조사하는 목적이라 UI 기본값을 섞지 않음)
- **인코딩**: UTF-8 BOM 포함(엑셀 한글 깨짐 방지) — 업로드 파서도 "BOM 포함/미포함 모두 허용"이라 그대로 재업로드 가능
- **CSV 이스케이프**: 필드에 쉼표/따옴표/줄바꿈이 있으면 큰따옴표로 감싸고 내부 따옴표는 두 번 반복(표준 CSV 규칙) — 골프장명에 쉼표가 들어가는 경우는 실데이터에서 아직 확인 못했으나 방어적으로 처리
- **진입점**: 11번 화면(`/admin/golf-courses`)에 "⇩ CSV 내보내기" 버튼 추가(업로드 버튼 바로 아래)
- **미검증**: 샌드박스가 로컬 DB에 접근할 수 없어 실제 다운로드 파일 내용은 미확인 — 사용자가 로컬에서 버튼 클릭 후 확인 필요

## 향후 보완 후보 (미확정, 구현 시점에 재검토)

- 골프장명 대신 `externalOrgCd`/`externalMngNo`(공공데이터 식별자) 기준 매칭 옵션 — 동명 골프장 구분 문제 해결
- 업로드 전 미리보기(실제 반영 전 검증 결과만 먼저 보여주기)
- 샘플 CSV 다운로드 엔드포인트 (`GET /api/admin/golf-courses/upload/sample`)
- CSV로 루프를 자동 생성할 때 오탈자로 의도치 않은 루프가 늘어날 수 있음 — 신규 루프 생성 건은 별도로 강조 표시해 관리자가 확인하게 하는 안 검토
